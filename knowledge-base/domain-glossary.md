# 金融領域術語表（Domain Glossary）

> 地位：**語義基石**。確保 AI 在分析 Bug 時不會混淆「清算」與「結算」，或在描述「冪等」時誤用概念。

---

| 術語 (CN) | 術語 (EN) | 定義與偵測重點 | 易混淆點辨析 |
| :--- | :--- | :--- | :--- |
| **清算** | Clearing | 交易後的計算過程。包括交易指令核對、費用計算、淨額轧差。 | **vs 結算**：清算不涉及資金移動，僅是「算帳」。 |
| **結算** | Settlement | 資金實際在帳戶/機構間轉移並入帳的終結過程。 | **重點**：結算失敗會導致帳實不符（Persistence Consistency）。 |
| **冪等** | Idempotency | 同一操作重複執行多次與執行一次的結果相同。 | **金融特有**：不僅是 HTTP 200，而是防範重複扣款。 |
| **對帳** | Reconciliation | 內部總帳、分帳與外部流水間的一致性校驗。 | **偵測**：若代碼中缺乏對帳鉤子，屬高危設計缺陷。 |
| **沖正** | Reversal | 因超時或技術錯誤導致交易狀態不明時，發起的技術撤回。 | **vs 退款**：沖正是為了修正異常，退款是業務行為。 |
| **退款** | Refund | 業務成功後，由用戶或商戶發起的資金返還請求。 | **風險**：退款路徑常缺越權校驗或重複退款檢查。 |
| **在途資金** | Float | 交易發起後已扣款，但尚未抵達接收方，暫存在中間科目的資金。 | **監控**：在途資金異常掛帳是洗錢或資金挪用的訊號。 |
| **轧帳** | Cut-off | 系統切換交易日、凍結當前週期流水進行結算的時刻。 | **風險**：Cut-off 邊界處的併發請求易產生雙倍入帳。 |
| **擔保交收** | Escrow | 資金先由第三方託管，待特定條件（如物流簽收）達成後釋放。 | **不變量**：未簽收前資金不得進入商戶可用餘額。 |
| **總帳與流水** | Ledger vs Journal | 總帳記錄靜態餘額（Balance），流水記錄動態事件（Event）。 | **校驗**：$\sum \text{Journal} = \Delta \text{Ledger}$ 是核心守恆律。 |
| **代收代付** | Collection/Payout | 代表客戶處理進項或出項資金的行為。 | **風險**：資金池混淆，缺乏歸屬權分離。 |
